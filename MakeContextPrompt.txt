Analyze this entire project codebase thoroughly, then create a file called
`CONTEXT.md` in the project root.

reading this one file should be able to understand the project completely
and start contributing immediately with zero extra explanation.

The file must include:

## Project Overview
- What this project is and its core purpose in 2-3 sentences
- Target platform and how it's run or deployed

## File & Folder Structure
- A concise map of what lives where
- What the most important files are and why
- Any non-obvious folder purposes
- Folders and files to ignore and never read
- One line description for every script explaining what it is and does

## Architecture & Patterns
- How the codebase is organized and why
- Key base classes, shared systems or utilities everything else depends on
- State management approach
- Naming conventions used consistently throughout
